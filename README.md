# openclaw-demo

A Docker Compose deployment configuration for [OpenClaw](https://openclaw.dev) — an AI gateway that proxies requests to LLM providers (Anthropic, etc.) and exposes a Web UI, deployed behind a Traefik reverse proxy with TLS.

## Overview

This repository contains the infrastructure configuration to self-host an OpenClaw gateway instance. It includes:

- **Docker Compose** service definition for the `openclaw-gateway` container
- **Traefik dynamic configuration** for HTTPS routing
- **Environment variable template** for secrets and settings
- **Telegram bot integration** for interacting with the agent via Telegram

## Prerequisites

- Docker and Docker Compose installed on the host
- A running [Traefik](https://traefik.io/) instance with:
  - An external Docker network named `traefik-public`
  - A `websecure` entrypoint configured
  - TLS certificates available at `/certs/cert.pem` and `/certs/key.pem`
- An [Anthropic API key](https://console.anthropic.com/)
- A Telegram bot token from [@BotFather](https://t.me/BotFather)

## Setup

### 1. Clone the repo

```bash
git clone <this-repo>
cd openclaw-demo
```

### 2. Configure environment variables

Copy the example environment file and fill in your values:

```bash
cp .env.example .env
```

Edit `.env`:

| Variable | Description |
|---|---|
| `OPENCLAW_GATEWAY_TOKEN` | Secure token for the Web UI (generate with `openssl rand -hex 32`) |
| `ANTHROPIC_API_KEY` | Your Anthropic API key |
| `TZ` | Your timezone (e.g. `Europe/Berlin`) |

### 3. Configure OpenClaw

```bash
mkdir -p data/credentials data/identity
cp data-examples/openclaw.json data/openclaw.json
cp data-examples/credentials/telegram-allowFrom.json data/credentials/telegram-allowFrom.json
cp data-examples/credentials/telegram-pairing.json data/credentials/telegram-pairing.json
```

Edit `data/openclaw.json` and replace:

| Placeholder | Value |
|---|---|
| `YOUR_TELEGRAM_BOT_TOKEN` | Token from @BotFather |
| `YOUR_GATEWAY_AUTH_TOKEN` | Same value as `OPENCLAW_GATEWAY_TOKEN` in `.env` |
| `https://your-domain.example.com` | Your actual domain |

Edit `data/credentials/telegram-allowFrom.json` and replace `YOUR_TELEGRAM_USER_ID` with your Telegram user ID (find it via [@userinfobot](https://t.me/userinfobot)).

> **Note:** `data/identity/device.json` and `data/identity/device-auth.json` are auto-generated by OpenClaw on first run — do not create them manually.

### 4. Configure Traefik routing

Copy `traefik-dynamic-openclaw.yml` to your Traefik dynamic configuration directory:

```bash
cp traefik-dynamic-openclaw.yml ~/traefik/dynamic/openclaw.yml
```

Update the `Host` rule to match your domain:

```yaml
rule: "Host(`your-domain.example.com`)"
```

### 5. Start the service

```bash
docker compose up -d
```

Check logs:

```bash
docker compose logs -f
```

The gateway will be available at `https://your-domain.example.com` once Traefik picks up the dynamic configuration.

## Project Structure

```
.
├── .env.example                        # Environment variable template
├── .gitignore                          # Ignores .env, data/, workspace/
├── docker-compose.yml                  # OpenClaw gateway service definition
├── traefik-dynamic-openclaw.yml        # Traefik HTTP router and service config
├── data/                               # Runtime data (gitignored — contains secrets)
│   ├── openclaw.json                   # Main config
│   ├── credentials/                    # Telegram credentials
│   └── identity/                      # Auto-generated device keys & tokens
└── data-examples/                      # Safe-to-commit templates for data/
```

Runtime directories (created automatically, git-ignored):

- `data/` — persistent OpenClaw configuration and state (copy from `data-examples/` to get started)
- `workspace/` — OpenClaw workspace files

## Health Check

The container exposes a health endpoint at `http://localhost:18789/health`. Docker checks it every 30 seconds (3 retries, 15s start period).

## Stopping the Service

```bash
docker compose down
```
